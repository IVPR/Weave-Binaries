Classpublic class AbstractDataSource

Implements IDataSource, IDisposableObject
Subclasses AbstractDataSource_old, AnnotationDataSource, CensusDataSource, CKANDataSource, CSVDataSource, DBFDataSource, ForeignDataMappingTransform, FREDDataSource, GeoJSONDataSource, GraphMLDataSource, GroupedDataTransform, HealthIndicatorsDataSource, PartitionDataTransform, SocrataDataSource, WeaveAnalystDataSource

This is a base class to make it easier to develope a new class that implements IDataSource. Classes that extend AbstractDataSource should implement the following methods: getHierarchyRoot, generateHierarchyNode, requestColumnFromSource

initializationCompleteproperty 
initializationComplete:Boolean  [read-only]

Classes that extend AbstractDataSource can define their own replacement for this function. All column requests will be delayed as long as this accessor function returns false. The default behavior is to return false during the time between a change in the session state and when initialize() is called.

findHierarchyNode()method 
public function findHierarchyNode(metadata:Object):IWeaveTreeNode

The default implementation of this function calls generateHierarchyNode(metadata) and then traverses the _rootNode to find a matching node. This function should be overridden if the hierachy is not known completely, since this may result in traversing the entire hierarchy, causing many remote procedure calls if the hierarchy is stored remotely. Finds the hierarchy node that corresponds to a set of metadata, or null if there is no such node.

Parameters

metadata:Object — Metadata used to identify a node in the hierarchy, which may or may not reference a column.

Returns
IWeaveTreeNode — The hierarchy node corresponding to the metadata or null if there is no corresponding node.

getHierarchyRoot()method 
public function getHierarchyRoot():IWeaveTreeNode

This function must be implemented by classes that extend AbstractDataSource. This function should set _rootNode if it is null, which may happen from calling refreshHierarchy(). Gets the root node of the attribute hierarchy, which should have descendant nodes that implement IColumnReference.

handlePendingColumnRequest()method 
protected function handlePendingColumnRequest(column:ProxyColumn):void

This function will call requestColumnFromSource() if initializationComplete==true. Otherwise, it will delay the column request again. This function may be overridden by classes that extend AbstractDataSource. However, if the extending class decides it wants to call requestColumnFromSource() for the pending column, it is recommended to call super.handlePendingColumnRequest() instead.

Parameters

column:ProxyColumn — The request that needs to be handled.
